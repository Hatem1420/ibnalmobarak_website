# منصة البرامج المساندة — دليل الإعداد

تم تقسيم المشروع إلى وحدات (modules) منظمة على نمط OOP بدلاً من ملف واحد ضخم،
بحيث يسهل صيانته وتوسيعه مستقبلاً:

```
quran-platform/
├── index.html                    ← الصفحة الرئيسية (الهيكل فقط)
├── css/
│   └── styles.css                ← كل التنسيقات (لم تتغيّر)
├── js/
│   ├── config.js                 ← الملف الوحيد الذي تعدّله للربط بحساباتك
│   ├── main.js                   ← نقطة تشغيل الموقع (bootstrap)
│   ├── core/
│   │   └── App.js                ← الصنف المنسّق: يربط الخدمات والحالة والمكوّنات ببعضها
│   ├── state/
│   │   └── AppState.js           ← مخزن الحالة المشتركة (البرامج، شرائح المتصفح، المرحلة الحالية)
│   ├── services/                 ← طبقة البيانات (بدون أي واجهة مستخدم)
│   │   ├── SheetsService.js      ← الاتصال بـ Google Sheets وتحويل الصفوف إلى بيانات برامج
│   │   ├── CsvParser.js          ← محلّل CSV مستقل قابل لإعادة الاستخدام
│   │   └── PreviewResolver.js    ← يكتشف نوع رابط المعاينة (يوتيوب / Drive / صورة)
│   ├── components/                ← كل جزء واجهة كصنف مستقل
│   │   ├── ProgramCards.js       ← شبكة بطاقات البرامج في الرئيسية
│   │   ├── HomeSwiper.js         ← شريط الصور القابل للسحب في الرئيسية
│   │   ├── StageTabs.js          ← تبويبات المراحل والبطاقات المصغّرة
│   │   ├── ProgramModal.js       ← نافذة تفاصيل البرنامج
│   │   ├── Lightbox.js           ← عرض الصورة بملء الشاشة
│   │   ├── Navigation.js         ← التنقل بين الأقسام (الرئيسية/المراحل/الدرجات)
│   │   ├── ThemeToggle.js        ← تبديل الوضع الليلي/النهاري
│   │   ├── LookerEmbed.js        ← تضمين تقرير Looker Studio
│   │   └── StatusBanner.js       ← شريط حالة تحميل البيانات
│   └── utils/
│       ├── dom.js                ← دوال مساعدة عامة (escapeHtml، اختصارات DOM)
│       └── constants.js          ← جداول التسميات والأيقونات الثابتة
└── sheet-template/
    ├── Programs_template.csv     ← نموذج جاهز لاستيراده إلى Google Sheets
    ├── Stats_template.csv        ← نموذج تبويب الإحصائيات (اختياري)
    └── Swiper_template.csv       ← نموذج تبويب صور المتصفح في الرئيسية (اختياري)
```

**لماذا هذا التقسيم؟** كل صنف (class) له مسؤولية واحدة واضحة: `services/` يتعامل
فقط مع جلب/تحويل البيانات، `components/` يتعامل فقط مع عرض واجهة المستخدم
والتفاعل معها، و`core/App.js` هو الوحيد الذي يعرف عن الجميع ويربطهم. هذا يجعل
إضافة ميزة جديدة (مثال: تبويب إحصائيات جديد، أو نوع معاينة جديد) أمراً معزولاً
لا يتطلب لمس بقية الكود، وهو نمط شائع في مشاريع الويب الصغيرة والمتوسطة القابلة
للنمو.

> **ملاحظة تقنية:** ملفات `js/` تُحمَّل الآن كوحدات ES Modules (`type="module"`)،
> لذا يجب فتح الموقع عبر خادم ويب (مثل GitHub Pages أو Netlify أو
> `python -m http.server` أثناء التطوير المحلي) وليس بفتح `index.html` مباشرة
> من نظام الملفات (`file://`)، لأن المتصفحات تمنع تحميل الوحدات من `file://` لأسباب أمنية.
> هذا لا يؤثر على النشر النهائي المذكور في القسم 5 أدناه.

كل البرامج الآن تُقرأ مباشرة من Google Sheets بدل أن تكون مكتوبة داخل الكود، ولوحة
الدرجات أصبحت تعرض تقرير Looker Studio الفعلي بدل الإطار الوهمي السابق.

---

## 1) ربط Google Sheets

### أ. أنشئ الشيت
1. أنشئ Google Sheet جديد.
2. أنشئ التبويبات التالية (Sheet tabs):
   - **Programs** — استورد إليه `sheet-template/Programs_template.csv`
     (ملف ← استيراد ← رفع ← استبدال بيانات الورقة الحالية).
   - **Stats** (اختياري) — استورد إليه `sheet-template/Stats_template.csv`.
   - **Swiper** (اختياري) — استورد إليه `sheet-template/Swiper_template.csv`
     لتخصيص صور شريط التصفح في الرئيسية بشكل مستقل عن البرامج (انظر القسم 3).

### ب. أعمدة تبويب Programs

| العمود | الوصف | القيم الممكنة |
|---|---|---|
| `id` | معرّف فريد للبرنامج | نص حر، بدون تكرار |
| `title` | عنوان البرنامج | نص |
| `type` | تصنيف البرنامج | `cultural` ثقافي / `tech` تقني / `edu` تعليمي / `moral` تربوي |
| `stage` | المرحلة الدراسية | `all` (يظهر في الرئيسية لجميع المراحل) أو `secondary` / `middle` / `upper-primary` / `lower-primary` |
| `timing` | حالة البرنامج الزمنية | `current` جارٍ / `upcoming` قادم / `past` سابق |
| `is_new` | يضيف وسم "جديد" في الرئيسية | `TRUE` أو `FALSE` |
| `date` | تاريخ العرض | أي نص، مثل `1445/7/1` |
| `description` | وصف البرنامج | نص |
| `conditions` | شروط الاشتراك | افصل بين الشروط بفاصلة منقوطة `;` |
| `preview_url` | معاينة تعريفية (اختياري) | رابط يوتيوب، أو رابط صورة Google Drive مُشارك، أو رابط صورة مباشر (انظر القسم 3) |

اترك `preview_url` فارغاً للبرامج التي لا تحتاج معاينة.

### ج. شارك الشيت بشكل عام
الموقع يقرأ البيانات بدون مفتاح API، لذلك:
1. اضغط **مشاركة (Share)** أعلى يمين الشيت.
2. غيّر الإعداد إلى **"أي شخص لديه الرابط" (Anyone with the link)** بصلاحية **عرض (Viewer)**.

### د. احصل على المعرّفات وضعها في `js/config.js`
- **GOOGLE_SHEET_ID**: من رابط الشيت، الجزء بين `/d/` و `/edit`
  مثال: `docs.google.com/spreadsheets/d/`**`1AbCDeFG...`**`/edit`
- **PROGRAMS_GID**، **STATS_GID**، **SWIPER_GID**: افتح كل تبويب وانظر إلى الرقم
  بعد `gid=` في الرابط (تبويب الشيت الأول عادة `gid=0`).

افتح `js/config.js` وعدّل:
```js
GOOGLE_SHEET_ID: "ضع_معرّف_الشيت_هنا",
SHEETS: {
  PROGRAMS_GID: "0",
  STATS_GID: "123456789",
  SWIPER_GID: "987654321", // اتركه فارغاً "" إن لم تُنشئ تبويب Swiper
},
```

أي تعديل أو إضافة برنامج تقوم به داخل Google Sheets سيظهر مباشرة على الموقع
(يُعاد تحميل البيانات تلقائياً كل بضع دقائق حسب `AUTO_REFRESH_MINUTES`،
وكذلك عند فتح الصفحة من جديد).

## 2) ربط لوحة الدرجات بـ Looker Studio

1. افتح تقريرك في **Looker Studio**.
2. من القائمة اضغط **مشاركة (Share) ← تضمين تقرير (Embed report)**.
3. فعّل خيار التضمين وانسخ الرابط الموجود داخل `src="..."`.
4. الصق الرابط في `js/config.js`:
```js
LOOKER_STUDIO_EMBED_URL: "https://lookerstudio.google.com/embed/reporting/REPORT_ID/page/PAGE_ID",
```

ملاحظة: تأكد أن مشاركة التقرير في Looker Studio مفعّلة لمن يملك الرابط، وإلا
سيظهر للزوار طلب تسجيل دخول داخل الإطار.

مربعات الإحصائيات الأربعة (متوسط الدرجات، إجمالي الطلاب، إلخ) تُقرأ تلقائياً
من تبويب **Stats** إن وُجد، وإلا تبقى القيم الافتراضية الظاهرة في الصفحة.

---

## 3) معاينات البرامج (preview_url) — فيديو أو صورة

عمود preview_url في تبويب Programs يقبل ثلاثة أنواع من الروابط، ويكتشف
الموقع النوع تلقائياً:

| النوع | مثال | طريقة العرض |
|---|---|---|
| فيديو يوتيوب | رابط youtu.be أو watch؟v= أو embed أو shorts، أو المعرّف المجرد (11 حرف) | مشغّل فيديو مضمّن (responsive) داخل تفاصيل البرنامج |
| صورة Google Drive | رابط مشاركة من نوع file/d/FILE_ID/view | صورة ثابتة (يُشترط أن يكون الملف مُشاركاً لأي شخص لديه الرابط) |
| صورة مباشرة | أي رابط صورة ينتهي بامتداد صورة (jpg, png, webp...) أو رابط googleusercontent.com | صورة ثابتة تُستخدم كما هي |

عند وجود قيمة في preview_url، تظهر تلقائياً:
- علامة "فيديو" أو "صورة" الصغيرة على بطاقة البرنامج وبطاقاته المصغّرة.
- المعاينة نفسها (مشغّل الفيديو أو الصورة) أعلى تفاصيل البرنامج عند فتح المودال.

اترك preview_url فارغاً للبرامج التي لا تحتاج معاينة.

## 4) شريط صور الرئيسية (Home Swiper) من تبويب مستقل

الشريط المتحرك (Swiper) أعلى الصفحة الرئيسية يمكن ربطه بمصدر بيانات مستقل
عن البرامج، بحيث تتحكم بصوره ومحتواه دون التأثير على تبويب Programs.

### أعمدة تبويب Swiper (اختياري)

| العمود | الوصف |
|---|---|
| image_url | نفس أنواع الروابط المدعومة في preview_url أعلاه (يوتيوب / Drive / صورة مباشرة) |
| title | عنوان الشريحة |
| date | تاريخ يظهر أسفل العنوان (اختياري) |
| time | وقت يظهر أسفل العنوان (اختياري) |
| points | عدد النقاط يظهر أسفل العنوان (اختياري) |
| program_id | إن أردت أن يفتح الضغط على الشريحة مودال برنامج معيّن، ضع هنا نفس id الخاص به من تبويب Programs. اتركه فارغاً لشريحة معلوماتية فقط بدون ضغط |

**آلية العمل:**
- إذا كان SWIPER_GID مضبوطاً في config.js والتبويب يحتوي بيانات صالحة،
  يعرض الشريط هذه الشرائح بترتيبها في الشيت.
- إذا لم يُضبط SWIPER_GID (تركته فارغاً)، أو كان التبويب فارغاً/غير
  متاح، يعود الموقع تلقائياً لعرض البرامج المعلّمة is_new=TRUE وصورها
  من عمود preview_url كما كان سابقاً — لا حاجة لأي إعداد إضافي إن كنت لا
  تريد هذه الميزة.

---

## 5) النشر

المشروع عبارة عن ملفات ثابتة (HTML/CSS/JS) فقط، يمكنك رفعه على أي استضافة
ثابتة (GitHub Pages، Netlify، Vercel، أو أي خادم ويب عادي) دون الحاجة لأي
خادم خلفي (backend) أو بناء (build step).

تأكد فقط من رفع المجلد بالكامل بنفس البنية (`css/`، `js/`، `index.html`)
حتى تعمل الروابط النسبية بشكل صحيح.
